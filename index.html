关于HashMap的教训

1.hashcode()和equals() 这两个自带的判别函数并不是完全可靠的，如果这两哥们不能发挥作用，最好自己写一个

2.对于HashMap，如果没有特别的操作，当录入两个Key值一样而Value值不一样的时，后录入的Value值会覆盖前一个录入的Value值，如果可以，最好避开这种情况，或者搞点骚操作来处理这种情况

肉毒杆菌

简介：严格厌氧、有鞭毛、无荚膜、产芽孢，广泛分布于土壤、海洋湖泊沉积物和家畜粪便中。
在厌氧环境中，肉毒杆菌可产生一种强烈的外毒素，即肉毒毒素。

病症：当婴儿感染时会导致“婴儿肉毒杆菌症”。
肉毒杆菌会在婴儿的肠胃道定殖并释放神经毒素，神经毒素会和肌细胞结合造成婴儿肌肉软瘫，呼吸困难。
即使是成人在感染后也会出现视线模糊、肌无力以及呼吸困难的情况。
若不及时就医可能导致死亡。

感染途径：感染肉毒杆菌十分罕见（在我国概率极低！）可能是由于吃下了肉毒杆菌的孢子，一个可能的来源是蜂蜜（未经过卫严格生安全处理的蜂蜜）同时孢子也可以存在于土壤甚至于灰尘里。
虽然几率很低，但在施工现场附近有可能会有被污染的土壤被翻动导致人吸入孢子

由肉毒杆菌导致的疾病若及时就医，可以根治

补充： 肉毒杆菌素 可以用于美容等方面（如除皱，改善脸型和腿型）。
但是单独使用 肉毒杆菌 祛除动态皱纹，是不可能有良好效果的，必须配合 玻尿酸 填充才能真正达到完美的除皱效果。
所以在注射肉毒杆菌的时候，一定要询问医师辅助填充治疗，避免上当受骗。
另外，使用 肉毒杆菌素 具有副作用，使用前需谨慎。

马蝇幼虫

简介： 成虫为昆虫，双翅目。因为其独特的繁殖方式而闻名。马蝇会把卵产在其他动物的身上（包括人类）

病症： 刺痛，寄生部位周围会出现炎症、感染、化脓，在孵化后期甚至可以感受到幼虫在体内爬行。
这是因为马蝇幼虫以宿主的组织和血液为食，为了不被宿主轻易排出体外，身上长有倒刺，在宿主体内时会将倒刺刺入宿主的组织用来固定自身和在宿主体内前进
幼虫通常并不会致命，但是十分麻烦并且会给宿主带来巨大的痛苦，知道幼虫孵化完成

感染途径： 蚊虫，动物（如牛、马）或接触马蝇的卵
在野外，并且是炎热且并不太卫生的区域就有可能出现马蝇，存在被感染的可能性

被马蝇幼虫寄生后可以到医院取出或自行将幼虫取出（极度不推荐，除非你是兰博）
实在不行，你可以等到幼虫孵化完成自己从你的身体里面钻出来，当然很痛，当然事后要去医院消毒

补充： 马蝇幼虫通常会在手臂、头皮的皮肤下孵化，但也有在蛋蛋里面孵化的病例。
最好的防护方法就是穿上长袖的衣服，避免虫子蛰咬。

利什曼原虫

简介：泛指利什曼虫的原虫，是一种会引起各种疾病的寄生虫，主要以沙蝇为媒介，感染对象从兔子之类的小动物到大型犬科无所不及（当然，包括人类）
在我国流行的利什曼原虫大都为杜氏利什曼原虫（可以引起黑热病）

病症： 利什曼原虫可以导致许多病症。
在我国，杜氏利什曼原虫可以引起黑热病（黑热病有多危险大家都清楚）；
在热带地区，利什曼原虫可以导致热带疮，患者皮肤会出现囊肿并在感染加深后开始溃烂（这是因为利什曼原虫在你体内开始聚餐，把你的组织当作食物进食的结果）
另外，利什曼原虫还可以导致发热，贫血以及淋巴结肿大等病症

感染途径： 利什曼原虫以沙蝇作为媒介来祸害各种动物，尤其是在热带地区，如果被带有利什曼原虫的沙蝇叮咬相对容易感染上利什曼原虫，但相对的，人类在不断对利什曼原虫进行研究，若就医及时，可及时治愈
另外，在我国，关于杜氏利什曼原虫导致的黑热病已经可以控制，可以做到及时发现及时治愈

补充： 由利什曼原虫造成的疾病由病患自身的免疫系统自行击退，但需要注意的是，若是被利什曼原虫感染的同时存在并发症视情况需要药物进行治疗
利什曼原虫一般会在人体内的巨噬细胞中寄生和繁殖，期间如果被免疫系统检测，身体会精选复杂的免疫现象（简而言之，就是浑身难受！！！）

最长回文串的直球解法(JAVA)

不得不说，你在尝试暴力解法的时候真的很直男

好吧，先把题目贴出来

给你一份『词汇表』（字符串数组） words 和一张『字母表』（字符串） chars。
假如你可以用 chars 中的『字母』（字符）拼写出 words 中的某个『单词』（字符串），那么我们就认为你掌握了这个单词。
注意：每次拼写时，chars 中的每个字母都只能用一次。
返回词汇表 words 中你掌握的所有单词的 长度之和。

我知道，绝对有人看到这个题目最先想的就是把这个字符串拆掉

没错我也是，嘿嘿（。。）

结果往往是一个百把行的代码还出现了Bug还不知道那里错了，难受 >_<

好吧不废话了，来点实在的

public static void main(String[] args) {
    String s = new String();
    s = "Bansanana";
    //提交算法
    final int N = 26;
    int[] Capital = new int[N]; //放大写字母
    int[] Lower = new int[N];   //放小写字母
    char[] Loge = new char[s.length()];
    int counst = 0,Fia = 0;

    Loge = s.toCharArray();  //拆分字符串
    
    //区分大小写，分开处理
    for(int i=0;i<s.length();i++){
        if(Loge[i] >= 'A' && Loge[i] <= 'Z')
            Capital[Loge[i] - 'A']++;  //统计出现次数
        if(Loge[i] >= 'a' && Loge[i] <= 'z')
            Lower[Loge[i] - 'a']++; 
    }

    //判断字符串的长度，分奇偶情况
    if(s.length() % 2 == 0){
    //偶数情况
        for(int i=0;i<N;i++){
            if(Capital[i] >= 2)
                counst = counst + Capital[i] / 2;  //统计成对的字符
        }
        for(int i=0;i<N;i++){
            if(Lower[i] >= 2)
                counst = counst + Lower[i] / 2;
        }

        //偶数情况下，如果成对的字符数*2和字符串的长度相对，那么这个字符串本身就是最长的回文串
        //否则，就说明这个字符串中有至少两个单独字符，随便拿一个出来组成回文串即可
        if(counst * 2 == s.length())
            Fia = s.length();
        else
            Fia = counst * 2 + 1;
    }
    else{
    //奇数情况
        //这一段的操作和上面一模一样的，可以直接烤过来
        for(int i=0;i<N;i++){
            if(Capital[i] >= 2)
                counst = counst + Capital[i] / 2;
        }
        for(int i=0;i<N;i++){
            if(Lower[i] >= 2)
                counst = counst + Lower[i] / 2;
        }
        //奇数情况从这里开始不同了
        //在奇数情况下，至少有一个字符注孤生
        //当只有一个单独字符时，说明这个字符串本身就是最长回文串（单独字符做中间字符）
        //否则，这个字符串就会有至少三个字符是单独字符，随便抽一个单独字符当中间字符即可构成回文串
        if(counst * 2 + 1 == s.length())
            Fia = s.length();
        else
            Fia = counst * 2 + 1;
    }

    System.out.println("\n");
    System.out.println(Fia);
}

}

我知道有很多人被那个超长的例子给拦住了，有相同经历的在心中扣个一OK？

大概解释一下思路

用两个数组分别存储大写字母和小写字母的出现次数，每个字母的出现次数默认为0，如果出现一次就在相应位置上加1

最后在确认长度时，只要将成对的字母提取出来，再根据总长度的奇偶情况判断即可求出题目所求
（具体操作写在代码注释里面）

总之，这个题目并不是要我们去拼合字符串而是需要我们去统计出成对的字符，回文串的长度只是一个顺带的操作
